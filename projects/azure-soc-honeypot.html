Azure SOC Honeypot and Attack Map Lab

SOC Lab • Azure VM • Microsoft Sentinel • KQL • Threat Detection

This project involved building a full Windows honeypot in Azure, opening it to the internet, and capturing real-world attack traffic. By intentionally exposing the VM with an “allow all inbound” Network Security Group rule and disabling Windows Firewall inside the host, global attackers were able to reach the system and generate live authentication attempts.

SecurityEvent logs were forwarded using the Azure Monitor Agent (AMA) to a Log Analytics Workspace connected to Microsoft Sentinel. Inside Sentinel, I created KQL queries to filter failed logons (Event ID 4625), extract attacker IP addresses, and enrich those IPs using a custom Sentinel Watchlist containing geolocation data.

The culmination of the lab was an interactive world attack map built within a Sentinel Workbook. This map dynamically visualized attacker locations based on real-time honeypot logs, allowing the identification of high-frequency attacker countries and malicious IP ranges.

What I Built

  • Deployed a Windows 10 honeypot VM inside Azure
  • Configured an unrestricted NSG to intentionally expose the system
  • Disabled host firewall to allow global attack traffic
  • Forwarded logs to a Log Analytics Workspace via AMA
  • Connected workspace to Microsoft Sentinel for SIEM capabilities
  • Ingested and queried SecurityEvent logs using KQL
  • Uploaded a 54k-row geolocation CSV as a Sentinel Watchlist
  • Used ipv4_lookup() to enrich logs with country data
  • Built an interactive attack map dashboard inside Workbooks

Sample KQL Used

let GeoIPDB_FULL = _GetWatchlist("geoip");
let WindowsEvents = SecurityEvent
    | where EventID == 4625
    | where IpAddress != "" and isnotempty(IpAddress)
    | order by TimeGenerated desc
    | evaluate ipv4_lookup(GeoIPDB_FULL, IpAddress, network);

Key Insights

  • The honeypot received attacks within minutes of deployment.
  • Most attackers originated from VPS providers or botnet-controlled IP ranges.
  • Common username attempts included “admin”, “test”, “guest”, and “administrator”.
  • KQL log enrichment enabled immediate mapping of hostile activity by country.
  • Sentinel’s Workbook capabilities made large-scale attack visualization seamless.

Skills Demonstrated

Azure administration, NSG rules, Windows security logs, SIEM analysis, KQL querying, network traffic monitoring, log enrichment, attack visualization.

Summary

This lab replicates the daily workflow of a Security Operations Center (SOC) analyst: configuring log sources, analyzing authentication failures, enriching data, and visualizing attacker behavior. The hands-on experience strengthened my understanding of SIEM pipelines, cloud log ingestion, and threat intelligence analysis within Azure.
