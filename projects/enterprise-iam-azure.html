Executive Identity Scenario

An organization is migrating its identity infrastructure to Microsoft Entra ID. The identity administrator must onboard staff across IT, HR, and Finance departments, enable secure vendor access through B2B guest invitations, govern permissions through security group RBAC, investigate authentication activity through audit logs, and maintain security posture through MFA enforcement — all while enforcing Zero Trust principles. This lab simulates that full operational lifecycle end-to-end.

Phase 2 — User Lifecycle Onboarding

Six internal employees were provisioned across IT, HR, and Finance departments using the Entra ID new user creation workflow. Each account was assigned department metadata, a UPN, and a temporary password with forced change on first sign-in.

2.1 — New User Creation Workflow

Each user was created through the Entra ID new user wizard. Required fields including display name, user principal name, department, and usage location were completed. A temporary password was generated with a forced change enforced on first sign-in.

  • Display name, UPN, and department metadata assigned per user
  • Temporary password set with first-sign-in change enforced
  • Provisioning validated in directory post-creation

New user creation form showing required identity fields and initial account configuration.

Identity Architecture Model

Architecture Overview

The identity architecture follows a layered Zero Trust model. Users authenticate through Entra ID and are governed by department-aligned RBAC security groups. Guest identities are scoped through B2B collaboration with restricted access. All administrative and authentication activity flows into the audit telemetry pipeline, enabling continuous monitoring and compliance visibility.

Users → Entra ID Directory → RBAC Security Groups → Authentication Policies → Audit & Monitoring

IAM architecture diagram showing identity flow, group segmentation, and audit telemetry pipeline.

Enterprise applications panel showing user access assignments to integrated application resources.

Enterprise Access Layer

Application access is governed through Entra ID's enterprise application integration, ensuring users can only access applications their security group membership permits. This layer enforces the connection between RBAC group governance and actual resource access — completing the Zero Trust access control chain.

Key Insights

IDENTITY LIFECYCLE

Structured provisioning with enforced password policies and department metadata reduces onboarding error and access drift.

RBAC GOVERNANCE

Group-based access segmentation ensures permissions are governed at scale without per-user assignment complexity.

GUEST IDENTITY CONTROL

B2B collaboration with scoped guest access enables external contractor participation without exposing internal resources.

AUDIT VISIBILITY

Comprehensive audit log coverage across lifecycle events enables rapid investigation and compliance reporting.

Summary

This simulation demonstrates enterprise identity administration inside Microsoft Entra ID, replicating the full IAM operational lifecycle: user provisioning across IT, HR, and Finance departments; RBAC group governance with department-aligned security groups; guest identity management via B2B collaboration with scoped external access; tier-1 helpdesk workflows including admin-triggered password resets and SSPR portal review; authentication event monitoring through audit and activity logs; and security posture assessment via MFA enforcement, security info verification, and per-user security profiling — all performed within a Zero Trust architecture framework.
