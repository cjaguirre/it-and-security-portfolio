Enterprise Identity & Access Management Lab — Microsoft Entra ID

This project simulates a real enterprise identity environment using Microsoft Entra ID. The lab demonstrates identity lifecycle management, external collaboration, role-based access governance, helpdesk operations, authentication monitoring, audit investigation, and tenant security awareness in a Zero Trust cloud model.

Microsoft Entra ID IAM Simulation Zero Trust Helpdesk Workflow Audit & Monitoring

Business Scenario

Acting as a Cloud Identity Administrator, I simulated managing identities for a mid-size organization migrating to Microsoft Entra ID. Responsibilities included onboarding employees, handling guest access, governing role-based permissions, performing account recovery, monitoring authentication activity, and reviewing audit trails. This mirrors real IAM workflows found in enterprise helpdesk, security, and identity administration roles.

Lab Workflow

Phase 1 — Environment Setup

  • Step 1.1: Accessed Microsoft Entra Admin Center to validate tenant and identity control plane.
  • Step 1.2: Explored navigation surfaces including users, roles, monitoring, and security.

Demonstrates administrative awareness of enterprise IAM tooling.

Screenshots: Entra dashboard, navigation overview

Phase 2 — Identity Awareness

  • Reviewed tenant overview, directory structure, and domain metadata.
  • Documented cloud, on-prem, and hybrid identity models.

Shows conceptual understanding of identity architecture beyond interface usage.

Phase 3 — Internal Identity Lifecycle

  • Created six internal users representing IT, HR, and Finance roles.
  • Applied onboarding best practices including forced password resets.

Simulates enterprise employee onboarding workflows.

Phase 4 — External Collaboration

  • Invited a guest contractor using B2B collaboration.
  • Modified guest metadata to simulate lifecycle enrichment.

Demonstrates vendor and contractor identity management.

Phase 5 — Access Governance

  • Created department security groups.
  • Mapped users to RBAC structure.

Models enterprise role-based access segmentation.

Phase 6 — Helpdesk Operations

  • Password reset simulation.
  • Account disable and restore workflow.
  • User deletion and recovery.

Replicates Tier-1 identity support scenarios.

Phase 7 — Authentication Monitoring

  • Generated failed and successful login activity.
  • Reviewed sign-in telemetry.

Demonstrates authentication troubleshooting and visibility.

Phase 8 — Audit Investigation

  • Generated identity change events.
  • Analyzed audit logs and correlation metadata.

Highlights enterprise traceability and forensic awareness.

Phase 9 — Security Posture Review

  • Reviewed tenant secure score.
  • Observed conditional access structure.

Introduces Zero Trust security awareness.

Phase 10 — Architecture Documentation

Created conceptual diagram showing identity flow: Users → Entra → Groups → Authentication → Logs.
