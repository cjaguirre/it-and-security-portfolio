Phase 5 — Shared Access & PowerShell

The shared folder is where group-based permissions get proven in practice. A folder named corp_shared was created on DC01, shared on the network, and its NTFS permissions were assigned to the All security group — not individual users. Adding or removing someone from the group immediately changes what they can access.

Group-based folder access

  • • corp_shared folder created and shared on the network
  • • NTFS and share permissions assigned to the All security group
  • • Verified that membership changes immediately affect access
  • • Demonstrates scalable RBAC — manage the group, not individual ACEs

PowerShell — scripted admin

The same identity operations done in the GUI can be automated with PowerShell. These sample commands cover creating a user, adding them to a group, and resetting their password — the three most common helpdesk tasks in any AD environment.

Create & enable a new user

New-ADUser -Name "Test User" `
           -GivenName "Test" `
           -Surname "User" `
           -SamAccountName "test.user" `
           -UserPrincipalName "test.user@adlab.local" `
           -AccountPassword (Read-Host -AsSecureString "Password") `
           -ChangePasswordAtLogon $true `
           -Enabled $true

Group membership & password reset

# Add user to group
Add-ADGroupMember -Identity "All" `
                 -Members "test.user"

# Reset password, force change on next login
Set-ADAccountPassword -Identity "test.user" `
                      -Reset `
                      -NewPassword (Read-Host -AsSecureString)
Set-ADUser -Identity "test.user" `
           -ChangePasswordAtLogon $true

Directory Overview & Best Practices

adlab.local domain in ADUC — domain tree, default containers, and lab objects on DC01

Tier 1 — Identity Hygiene

Keep accounts, passwords, and groups clean from day one.

  • Strong credential policyEnforce password complexity and rotation, especially for privileged accounts.
  • Least privilege groupsAssign permissions through role-based groups — not per individual user.
  • Account lifecycleDisable or remove accounts immediately when users leave or change roles.

Tier 2 — Access Governance

Keep permissions auditable and easy to troubleshoot.

  • Group-based folder accessMap shared resources to security groups — adding/removing members is faster and cleaner than editing ACLs.
  • Consistent naming conventionsClear names for users, groups, and OUs make audits and troubleshooting far quicker.
  • Periodic access reviewsReview group membership and folder ACLs regularly to match actual usage.

Tier 3 — Operational Resilience

Build repeatable habits that scale.

  • DC backups & snapshotsSnapshot the domain controller before any major change — AD restores are painful without one.
  • Monitor logon failuresTrack failed logons, lockouts, and admin actions as early warning signals.
  • Scripted operationsPowerShell standardizes onboarding and offboarding — reduces errors and speeds up response.

  • • Clean OU design early on makes future group policy and troubleshooting much easier
  • • Running AD in Azure makes it safe to practice misconfigurations and fixes without touching production
