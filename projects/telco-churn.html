Telco Customer Churn Analysis

Analyzed telecom customer data to understand why people cancel service and built a model that predicts which customers are likely to leave so the business can act before they churn.

Churn Risk Analysis · Predictive Modeling Tools Used: Python, Pandas, Scikit-Learn, SHAP, Jupyter Notebook

Project Narrative

I used a telecom customer dataset to identify which customers stay, which ones leave, and what patterns predict churn. After cleaning and preparing the data, I converted text fields into numeric features and split the dataset into training and testing sets so the model could learn patterns without being biased.

I trained classification models and used SHAP explainability to see which factors influence churn the most. This makes the model transparent and gives a business team clear insights to guide retention strategies and focus on high-risk customers.

What I Built

  • A full churn-analysis workflow in Jupyter Notebook.
  • Data cleaning to fix billing issues and remove unusable rows.
  • Feature engineering to convert categories into model-ready inputs.
  • A balanced train/test split for fair evaluation.
  • Classification models (Logistic Regression + Random Forest)
  • Confusion matrix + classification report for performance.
  • SHAP explainability to highlight the strongest churn drivers.

2. Cleaning & Transformation

Fixing billing fields and removing unreliable rows

The TotalCharges column wasn’t always numeric — blank strings and invalid entries would break the model. Converting it to a numeric field forced those errors into NaN, making them easy to filter out. After that, incomplete rows were removed, and the customerID column was dropped since it doesn’t help predict churn.

The screenshot below shows the cleaning steps in the notebook and confirms that invalid billing entries were fixed, missing values were checked, and unnecessary columns were removed before modeling.

Cleaning logic applied to convert TotalCharges, drop incomplete rows, and remove non-predictive identifiers. 

                df["TotalCharges"] = pd.to_numeric(df["TotalCharges"], errors="coerce")
                df = df.dropna()
                df = df.drop(columns=["customerID"])

These steps ensure the model trains only on complete, meaningful features.

3. Feature Engineering & Split

Converting categories and preparing train/test sets

Most features in the dataset were text labels (like Yes, No, or plan names), so I converted all categorical columns into numeric values using one-hot encoding. This gives the model clean binary inputs without changing the meaning of the original data.

After encoding, I separated the churn label from the rest of the features and split the data into training and testing sets. Using stratify=y keeps the churn ratio consistent in both sets, preventing the model from training on an unbalanced sample. 

                df_encoded = pd.get_dummies(df, drop_first=True)
                
                X = df_encoded.drop("Churn_Yes", axis=1)
                Y = df_encoded["Churn_Yes"]
                
                X_train, X_test, y_train, y_test = train_test_split(
                    X, Y, test_size=0.2, random_state=42, stratify=Y
                )

4. Modeling & Evaluation

Training a Random Forest and checking performance

I trained a Random Forest classifier on the cleaned, encoded dataset to predict which customers are likely to churn. After fitting the model on the training split, I evaluated it on the test set to see how well it generalizes to unseen customers.

The evaluation uses a confusion matrix and classification report, which provide accuracy, precision, recall, and F1-scores for both churn and non-churn classes. 

                        model = RandomForestClassifier(
                            n_estimators=300,
                            max_depth=12,
                            random_state=42
                        )
                        
                        model.fit(X_train, y_train)
                        
                        y_pred = model.predict(X_test)
                        
                        print(confusion_matrix(y_test, y_pred))
                        print(classification_report(y_test, y_pred))

Evaluation results showing how accurately the model identifies churn and non-churn customers.

5. SHAP Explainability

Showing which features push churn risk up or down.

To understand why the model makes its predictions, I used SHAP values, which show how much each feature increases or decreases the likelihood of churn. This adds transparency to the Random Forest and helps highlight the main drivers behind customer behavior.

The SHAP summary plot ranks the most influential features—such as tenure, monthly charges, and contract type—and shows their overall impact on churn. This makes it easy to translate a technical model into clear, practical insight for both technical and non-technical audiences. 

                    explainer = shap.TreeExplainer(model)
                    shap_values = explainer.shap_values(X_test)
                    
                    shap.summary_plot(shap_values[1], X_test)
                    shap.summary_plot(shap_values[1], X_test, plot_type="bar")

SHAP summary plot showing which features have the strongest impact on churn predictions.
