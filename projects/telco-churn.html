Telco Customer Churn Analysis

Python • Machine Learning • Logistic Regression • Random Forest • SHAP Explainability • Data Cleaning • Feature Engineering

This project analyzes a telecommunications customer churn dataset and builds predictive machine learning models to identify which customers are most likely to leave. The analysis includes data cleaning, feature engineering, visualization, model training, evaluation, and SHAP explainability.

The objective was to produce an actionable churn model and provide business insights that help retention teams understand why customers churn and how to intervene.

Dataset Overview

The dataset contains 7,000+ customer records with variables related to:

  • Demographics (age, partner, dependents)
  • Service usage (internet, tech support, streaming)
  • Billing (monthly charges, payment method)
  • Contract information (month-to-month, yearly)

Target variable: Churn (Yes/No).

Data Cleaning & Preparation

Significant preprocessing steps were required to prepare the dataset:

  • Converted “TotalCharges” field to numeric
  • Handled missing values and blank string entries
  • Removed duplicate records
  • One-hot encoded all categorical variables
  • Scaled numerical features for logistic regression
df["TotalCharges"] = pd.to_numeric(df["TotalCharges"], errors="coerce")
df = df.dropna()
df = pd.get_dummies(df, drop_first=True)

Machine Learning Models

Two models were trained and compared:

  • Logistic Regression — baseline interpretable model
  • Random Forest Classifier — more complex, non-linear model

Model performance was evaluated using:

  • Confusion matrix
  • Accuracy
  • Precision / Recall
  • ROC-AUC
model = RandomForestClassifier(n_estimators=300, max_depth=12, random_state=42)
model.fit(X_train, y_train)
preds = model.predict(X_test)

Model Explainability (SHAP)

SHAP values were generated to understand which features most strongly influenced churn predictions. 

explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X)
shap.summary_plot(shap_values, X)

The SHAP summary plots revealed clear churn drivers.

Key Churn Drivers

  • Month-to-month contracts — highest predictor of churn
  • High monthly charges
  • Lack of fiber internet / poor service quality
  • No tech support or security add-ons
  • Electronic check billing — associated with higher churn

Key Outcomes

  • Built a reliable churn prediction model with strong classification metrics
  • Mapped out the top churn indicators using SHAP explainability
  • Created a business recommendation set for retention strategy
  • Produced visuals and analysis ready for executive reporting
  • Delivered an end-to-end machine learning workflow

Skills Demonstrated

Python, pandas, scikit-learn, SHAP interpretability, feature engineering, ML pipelines, data cleaning, exploratory analysis, KPI reporting.

Summary

This project showcases a complete machine learning workflow designed to help telecom companies identify at-risk customers and proactively reduce churn. By combining predictive modeling with explainability and business logic, the analysis transforms raw data into actionable retention insights.
