World Layoffs — SQL Data Cleaning

A full SQL cleaning and restructuring of a real-world global layoffs dataset. This project removed duplicates, standardized categories, transformed inconsistent fields, fixed dates, and built a clean staging pipeline ready for advanced analysis and global trend reporting.

Workforce Trends Data Standardization

Project Narrative

This dataset started as a messy global collection of layoff reports: inconsistent company names, misspelled industries, invalid dates, blank fields, duplicates, and contradictory entries.

Using MySQL, I rebuilt the data pipeline into a clean, trusted staging environment optimized for exploratory analysis, time-series reporting, and global workforce insights.

What I Built

  • Staging pipeline: raw → staging → cleaned final table.
  • Removed duplicates using ROW_NUMBER() window functions.
  • Standardized company, industry, and country fields.
  • Converted date text into proper DATE fields.
  • Handled missing values with conditional logic + self-joins.
  • Reshaped data for global analytics (year, quarter, categories).

Dataset Overview

The original dataset included layoffs across industries, companies, and countries, but contained:

  • Duplicate entries
  • Inconsistent formatting
  • Missing industry or layoff values
  • Invalid date formats
  • Extra whitespace and broken casing

Key Cleaning Steps

  • Duplicate removal using ROW_NUMBER().
  • Whitespace + casing cleanup.
  • Industry consolidation (e.g., “Crypto” variations).
  • Null handling using self-joins and conditional rules.
  • Date transformation with STR_TO_DATE().
  • Conversion of text columns to proper SQL types.
-- Remove duplicates
WITH cte AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY company, location, industry, date
           ) AS rn
    FROM layoffs_staging2
)
DELETE FROM layoffs_staging2 WHERE rn > 1;

-- Clean company names
UPDATE layoffs_staging2 SET company = TRIM(company);

-- Fix industry formatting
UPDATE layoffs_staging2 SET industry = 'Crypto' WHERE industry LIKE 'Crypto%';

-- Normalize country names
UPDATE layoffs_staging2
SET country = TRIM(TRAILING '.' FROM country)
WHERE country LIKE '%.';

Data Transformations

  • Extracted year + quarter from final layoff date.
  • Consolidated industries into standardized categories.
  • Prepared clean fields for dashboards and analytics.
ALTER TABLE layoffs_staging2 ADD COLUMN year INT;

UPDATE layoffs_staging2
SET year = EXTRACT(YEAR FROM date);
