World Layoffs — SQL Data Cleaning

A full SQL cleaning and transformation of a global layoffs dataset. This project corrected structural issues, standardized inconsistent fields, handled missing values, removed duplicates, and reshaped the data into a trusted source for global workforce analytics.

Data Standardization Workforce Trends

Project Narrative

The global layoffs dataset included thousands of records from companies across dozens of countries. However, the raw data was inconsistent — duplicated entries, messy company names, missing industries, invalid dates, and mismatched totals made it unreliable for analysis.

This project rebuilt the dataset using SQL cleaning techniques, creating a high-quality foundation ready for industry, geography, and time-series analysis.

What I Built

  • Removed duplicates using window functions and ranked CTEs.
  • Standardized company and industry naming conventions.
  • Transformed date strings into proper SQL date objects.
  • Cleaned whitespace, casing, and inconsistent category labels.
  • Created derived fields for year, quarter, and grouped industries.

Key Data Cleaning Steps

  • Removed duplicate entries using ROW_NUMBER() window functions.
  • Standardized company names (e.g., “Google LLC” → “Google”).
  • Filled missing industry data using intra-company mode logic.
  • Converted date strings to SQL DATE type.
  • Trimmed whitespace and fixed upper/lowercase inconsistencies.
  • Removed invalid or contradictory records.
-- Remove duplicates
WITH ranked AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY company, location, date
               ORDER BY layoffs DESC
           ) AS rn
    FROM layoffs_raw
)
DELETE FROM ranked WHERE rn > 1;

-- Standardizing industry values
UPDATE layoffs
SET industry = (
    SELECT mode() WITHIN GROUP (ORDER BY industry)
    FROM layoffs AS l2
    WHERE l2.company = layoffs.company
)
WHERE industry IS NULL;
