Landscape Database Revitalization for Business Insights

A complete rebuild of a fragmented landscaping company database into a clean, normalized, analytics-ready SQL Server system. Standardized the schema, enforced relational integrity, and built 30+ SQL queries to uncover customer trends, service patterns, and profitability insights.

Advanced Database Management · Spring 2023 Tools: SQL Server, SSMS, ERDPlus

Project Narrative

The original database suffered from duplicated client records, inconsistent service descriptions, and missing relational structure between projects, materials, and labor. Reporting was slow, unreliable, and required constant manual corrections.

I rebuilt the entire schema using an ERD-first approach, normalized the system, enforced keys, and authored queries that transform raw operational data into clear, decision-ready business insights.

ER Modeling & Database Design

Using ERDPlus, I modeled the core structure of the landscaping company. The ER model defines how clients, projects, materials, labor, and suppliers interact across each job.

  • Entities: Client, Project, Material, Labor, Supplier.
  • Attributes: names, contact info, labor hours, material costs, project dates.
  • Relationships: client–project (1:M), project–material (M:N), project–labor (M:N).
  • Refined cardinalities and dependencies to match real business workflows.

-- Client table
CREATE TABLE Client (
    ClientID   INT IDENTITY(1,1) PRIMARY KEY,
    ClientName VARCHAR(100) NOT NULL,
    Phone      VARCHAR(20),
    Email      VARCHAR(100),
    City       VARCHAR(50),
    StateCode  CHAR(2)
);

-- Project table
CREATE TABLE Project (
    ProjectID   INT IDENTITY(1,1) PRIMARY KEY,
    ClientID    INT NOT NULL,
    ProjectName VARCHAR(120) NOT NULL,
    StartDate   DATE,
    EndDate     DATE,
    Status      VARCHAR(30),
    CONSTRAINT FK_Project_Client
        FOREIGN KEY (ClientID) REFERENCES Client(ClientID)
);

Analytical Insight Queries

I developed 30+ queries to answer business-critical questions related to customer behavior, service popularity, labor efficiency, and material cost patterns.

  • Top revenue-generating services and seasonal demand changes.
  • Customer trends by region, frequency, project type, and lifetime value.
  • Profitability calculated from labor costs and material usage.
  • Supplier cost patterns and opportunities to reduce expenses.
-- Popular services by volume
SELECT      s.ServiceName,
            COUNT(*) AS ProjectCount
FROM        Project p
JOIN        ProjectService ps ON p.ProjectID = ps.ProjectID
JOIN        Service s ON ps.ServiceID = s.ServiceID
GROUP BY    s.ServiceName
ORDER BY    ProjectCount DESC;

-- Client profitability summary
SELECT      c.ClientName,
            p.ProjectName,
            SUM(jm.Quantity * m.UnitCost)     AS MaterialCost,
            SUM(l.HoursWorked * l.HourlyRate) AS LaborCost,
            p.BidAmount - (
                SUM(jm.Quantity * m.UnitCost) +
                SUM(l.HoursWorked * l.HourlyRate)
            ) AS EstimatedProfit
FROM        Project p
JOIN        Client c           ON p.ClientID = c.ClientID
JOIN        JobMaterials jm    ON p.ProjectID = jm.ProjectID
JOIN        Material m         ON jm.MaterialID = m.MaterialID
JOIN        LaborAssignment l  ON p.ProjectID = l.ProjectID
GROUP BY    c.ClientName, p.ProjectName, p.BidAmount
ORDER BY    EstimatedProfit DESC;
