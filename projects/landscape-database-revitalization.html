Landscape Database Revitalization for Business Insights

Rebuilt a fragmented landscaping company database into a normalized, analytics-ready SQL Server system. Standardized data, enforced relational integrity, and authored 30+ advanced SQL queries to surface customer trends, service popularity, and project profitability for decision-makers.

Advanced Database Management · Spring 2023 Tools: SQL Server, SSMS, ERDPlus

Project Narrative

A mid-sized landscaping company was running on a fragmented database: duplicated client records, inconsistent service descriptions, and poorly defined relationships between projects, labor, and materials. Reporting was slow, error-prone, and difficult to trust.

In this project, I redesigned the database in SQL Server using an ERD-driven approach, normalized core tables, enforced keys and constraints, and developed a suite of SQL queries that turn operational data into clear business insights related to customer trends, popular services, and project profitability.

ER Modeling & Database Design

The design process started in ERDPlus, where I modeled the core structure of the landscaping business. The ER model captured how clients, projects, materials, labor, and suppliers interact across jobs.

  • Entities: Client, Project, Material, Labor, Supplier.
  • Attributes: client details, project type, start/end dates, labor hours, material cost, supplier info.
  • Relationships: one-to-many (Client → Project), many-to-many (Project ↔ Material via junction table), and project–labor assignments.
  • Refined cardinalities and optionality to match real workflows and reporting requirements.

SQL Scripting & Schema Implementation

Using SQL Server Management Studio, I created the database, tables, and constraints to bring the ER model to life. The schema is designed for clarity, integrity, and performance. 

-- Create database
IF DB_ID('NBDDB_CTF') IS NOT NULL
    DROP DATABASE NBDDB_CTF;
GO

CREATE DATABASE NBDDB_CTF;
GO

USE NBDDB_CTF;
GO

-- Client table
CREATE TABLE Client (
    ClientID      INT IDENTITY(1,1) PRIMARY KEY,
    ClientName    VARCHAR(100) NOT NULL,
    Phone         VARCHAR(20),
    Email         VARCHAR(100),
    City          VARCHAR(50),
    StateCode     CHAR(2)
);

-- Project table
CREATE TABLE Project (
    ProjectID     INT IDENTITY(1,1) PRIMARY KEY,
    ClientID      INT NOT NULL,
    ProjectName   VARCHAR(120) NOT NULL,
    StartDate     DATE,
    EndDate       DATE,
    Status        VARCHAR(30),
    CONSTRAINT FK_Project_Client
        FOREIGN KEY (ClientID) REFERENCES Client(ClientID)
);

Business Insight Queries

Over 30 queries were developed to mirror instructor specifications (IMS-2 / IMS-3) and extend them into real business questions for the landscaping company.

  • Top revenue-generating services and seasonal demand patterns.
  • Customer trends by region, frequency, and spend.
  • Project profitability based on labor hours and material usage.
  • Supplier-level cost patterns and opportunities for margin improvement.
-- Example: Most popular services by project count
SELECT      s.ServiceName,
            COUNT(*) AS ProjectCount
FROM        Project p
JOIN        ProjectService ps ON p.ProjectID = ps.ProjectID
JOIN        Service s ON ps.ServiceID = s.ServiceID
GROUP BY    s.ServiceName
ORDER BY    ProjectCount DESC;

-- Example: Project profitability by client
SELECT      c.ClientName,
            p.ProjectName,
            SUM(jm.Quantity * m.UnitCost)     AS MaterialCost,
            SUM(l.HoursWorked * l.HourlyRate) AS LaborCost,
            p.BidAmount - (
                SUM(jm.Quantity * m.UnitCost) +
                SUM(l.HoursWorked * l.HourlyRate)
            ) AS EstimatedProfit
FROM        Project p
JOIN        Client c           ON p.ClientID = c.ClientID
JOIN        JobMaterials jm    ON p.ProjectID = jm.ProjectID
JOIN        Material m         ON jm.MaterialID = m.MaterialID
JOIN        LaborAssignment l  ON p.ProjectID = l.ProjectID
GROUP BY    c.ClientName, p.ProjectName, p.BidAmount
ORDER BY    EstimatedProfit DESC;
