Landscape Database Revitalization

SQL • Database normalization • Schema redesign • Business logic modeling

This project transformed a fragmented landscaping services database into a clean, normalized, analytics-ready system. The original dataset contained inconsistent fields, duplicated entries, and missing relational structure, which made billing, job tracking, and client reporting slow and unreliable.

The goal was to rebuild the entire backend: standardize service categories, normalize job and invoice tables, enforce relational integrity, and redesign the schema to support accurate reporting for operations and management. The result was a robust SQL database that improves efficiency, accuracy, and maintainability.

Problems Identified in the Original System

  • Client names stored inconsistently (misspellings, different formats)
  • Service descriptions varied widely and lacked standardization
  • No primary keys or relational constraints
  • Duplicate job entries with mismatched billing amounts
  • Invoices and jobs mixed in the same table (breaking normalization rules)
  • Hard-to-analyze fields like free-text notes and manual timestamps

Project Objectives

  • Redesign the database structure using proper relationships
  • Normalize tables to eliminate redundancy and inconsistencies
  • Create reliable keys to connect clients, jobs, materials, and invoices
  • Improve query performance and reporting clarity
  • Set the foundation for a future dashboard or CRM integration

New Database Schema

The database was rebuilt using a properly normalized structure consisting of the following tables:

  • Clients – business names, contact info, location
  • Services – standardized service categories, pricing models
  • Jobs – date, service type, assigned crew, materials used
  • Invoices – linked directly to jobs through foreign keys
  • Materials – mulch, soil, maintenance materials, etc.
  • JobMaterials (junction table) – quantity + cost per job

Sample SQL Improvements


-- Standardizing service categories
UPDATE Services
SET ServiceType = 'Lawn Maintenance'
WHERE ServiceType IN ('Lawn Care', 'Grass Cut', 'Cutting Grass');

-- Creating relational structure
ALTER TABLE Jobs
ADD CONSTRAINT fk_client
FOREIGN KEY (ClientID) REFERENCES Clients(ClientID);

-- Identifying duplicate jobs
SELECT JobID, ClientID, ServiceID, JobDate, COUNT(*)
FROM Jobs
GROUP BY JobID, ClientID, ServiceID, JobDate
HAVING COUNT(*) > 1;

Key Outcomes

  • 100% reduction in duplicate job entries
  • Clean, standardized service catalog enabling accurate billing
  • Faster SQL queries due to proper indexing and structure
  • Improved ability to track expenses and materials per job
  • Better reporting accuracy for monthly and seasonal performance
  • A fully scalable structure ready for dashboards or automation
