Bellabeat Case Study Fitbit Dataset R Programming Engagement Analytics

Bellabeat Smart Device Usage Analysis

Using 30 days of Fitbit activity and sleep data to uncover how wellness behavior reflects real smart-device engagement — helping Bellabeat tailor features, nudges, and marketing to match user habits.

Project Narrative

This project analyzes how users move, rest, and engage with wellness devices across a typical month. By merging daily steps, calories, intensity minutes, and sleep data, I developed a clear picture of when users are most active and when engagement drops.

These insights inform Bellabeat’s strategy for product features, engagement triggers, and wellness-focused messaging that meets users where they already are.

What I Built

  • End-to-end R data pipeline to clean and merge Fitbit tables.
  • Engineered features capturing daily activity + sleep behavior.
  • Exploratory visuals across steps, calories, and rest patterns.
  • Weekly engagement patterns tied to actionable recommendations.
  • Final dashboard summarizing all insights in a unified view.

Lab Breakdown

1. Dataset and Business Question

Fitbit data used as a proxy for Bellabeat users

The dataset consists of 30 Fitbit users over a 30-day window. It includes steps, calories, distance, intensity minutes, and sleep duration. The objective: determine what behavior patterns Bellabeat can leverage to increase retention and engagement.

Figure 1 — Merged activity + sleep dataset preview

2. Data Cleaning and Feature Engineering

Joined activity and sleep tables & engineered wellness metrics

  • Standardized date fields using mdy().
  • Joined activity + sleep tables on user and date.
  • Calculated total active minutes per day.
  • Derived weekday labels for behavior comparison.

Figure 2 — Final cleaned + feature engineered dataset (glimpse output)

# Merge and engineer core daily metrics
daily_activity <- read_csv("dailyActivity_merged.csv") %>% clean_names()
daily_sleep    <- read_csv("sleepDay_merged.csv") %>% clean_names()

daily_activity <- daily_activity %>%
  mutate(activity_date = mdy(activity_date))

daily_sleep <- daily_sleep %>%
  mutate(sleep_date = mdy(sleep_day))

merged <- daily_activity %>%
  inner_join(daily_sleep, by = c("id", "activity_date" = "sleep_date")) %>%
  mutate(
    day_of_week = wday(activity_date, label = TRUE),
    active_minutes_total =
      very_active_minutes + fairly_active_minutes + lightly_active_minutes
  )

3. Daily and Weekly Behavior Patterns

Weekly routines shape how users move and rest. Below are the most important engagement trends Bellabeat can use to customize wellness nudges.

  • Steps peak Tuesday–Thursday — workplace routines & gym habits.
  • Sleep clusters around 6–7.5 hours — below recommended 8 hours.
  • Weekends show sharp drops in calories and active minutes.
  • Users with consistent weekday routines engage more regularly.

These insights support better product design, personalized nudges, and long-term engagement programs aligned with real user behavior.
