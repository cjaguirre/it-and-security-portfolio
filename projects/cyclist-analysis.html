Cyclistic Rider Behavior Analysis

A year-long analysis of Chicago’s bike-share program comparing casual riders and annual members. Findings support the Google Data Analytics Capstone objective: informing strategies to convert high-usage casual riders into long-term members.

Rider Segmentation Usage Trends

Project Narrative

This analysis covers 12 months of Cyclistic bike-share data to understand differences between casual riders and annual members. Using R for wrangling and Tableau for visualization, I uncovered seasonal trends, weekday vs weekend behavior patterns, and ride duration differences.

Insights directly support marketing strategies focused on converting high-usage casual riders into long-term members, following the full Google Data Analytics Capstone workflow.

What I Built

  • Merged and cleaned 12 months of trip data in R using tidyverse.
  • Derived ride_length and day_of_week variables for segmentation.
  • Summarized statistical differences between rider types.
  • Created Tableau dashboards showing seasonal and hourly trends.
  • Generated clear recommendations to increase membership conversion.

Sample R Code

library(tidyverse)
library(lubridate)

# Load and combine monthly files
files <- list.files(pattern = "*.csv")
all_trips <- files %>% 
  map_df(read_csv) %>% 
  clean_names()

# Clean durations and remove bad data
all_trips <- all_trips %>% 
  mutate(
    started_at = ymd_hms(started_at),
    ended_at = ymd_hms(ended_at),
    ride_length = as.numeric(difftime(ended_at, started_at, units = "mins")),
    day_of_week = wday(started_at, label = TRUE)
  ) %>% 
  filter(ride_length > 1, !is.na(member_casual))

# Summary comparison
summary_stats <- all_trips %>% 
  group_by(member_casual) %>% 
  summarise(
    average_ride_length = mean(ride_length),
    ride_count = n()
  )
