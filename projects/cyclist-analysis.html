Project overview

I combined 12 months of Cyclistic trip data from Chicago to uncover behavioral differences between casual riders and annual members. Using R (tidyverse, lubridate, janitor, ggplot2) for preparation and analysis plus Tableau for visualization, I explored ride volume, duration and timing patterns across seasons and weekdays.

Insights directly support membership growth strategies by identifying how casual riders behave compared to members and where targeted promotions or in app nudges can convert high usage casuals into long term subscribers.

Key deliverables

  • Merged and cleaned 12 months of Divvy / Cyclistic trip data in R using tidyverse.
  • Engineered ride_length and day_of_week features for segmentation.
  • Computed summary statistics to compare ride behavior between rider types.
  • Developed ggplot2 visuals and a Tableau dashboard to surface trends and KPIs.
  • Produced concrete recommendations to increase membership conversion.

Lab Breakdown

Phase 1 — Importing and standardizing 12 months of Cyclistic data

The project begins by downloading 12 monthly CSVs from the Divvy / Cyclistic portal. These files contain millions of raw trip records with slightly different schemas over time. Using tidyverse, I imported and combined them into a single, clean dataframe ready for analysis.

1.1 — Package setup and environment readiness

RStudio was configured with tidyverse, lubridate, janitor, ggplot2 and readr to support the full workflow from import through visualization. This made the environment reproducible for anyone following the case study.

  • ● Installed and loaded tidyverse, lubridate, janitor, ggplot2 and readr.
  • ● Verified versions and library paths for clean reproducibility.
  • ● Created a dedicated project folder with raw and processed data directories.

RStudio console confirming successful installation of tidyverse, lubridate, janitor, ggplot2 and readr.

1.2 — Combining 12 CSVs into one all_trips dataframe

Using list.files(), map_df() and clean_names(), the 12 monthly CSVs were imported and standardized into one dataframe. Column names like ride_id, started_at, ended_at and member_casual were normalized for easier downstream work.

  • ● Collected file paths from the data_raw directory.
  • ● Used map_df(read_csv) to stack monthly datasets.
  • ● Cleaned column names with clean_names() for consistency.

RStudio environment after successfully importing and combining 12 months of Cyclistic trip data.

Phase 2 — Data cleaning and feature engineering

After import, the focus shifts to removing invalid records and creating features that explain rider behavior. Using lubridate and dplyr, I built ride length and weekday labels and filtered out rides that would distort the analysis.

2.1 — Cleaning timestamps and durations

Start and end timestamps were parsed into proper datetime objects, then used to calculate ride duration in minutes. Extremely short or negative rides were removed to avoid skewed averages.

  • ● Converted started_at and ended_at with ymd_hms().
  • ● Computed ride_length as a numeric difference in minutes.
  • ● Filtered out rides with duration less than or equal to one minute.

RStudio console showing mutation steps for ride length and weekday plus filters for invalid rides.

2.2 — Engineering behavioral features with lubridate

To support segmentation, day_of_week labels were created and later used to compare weekday commuting patterns with weekend leisure trips across rider types.

  • ● Derived day_of_week from started_at using wday(..., label = TRUE).
  • ● Classified rides into weekday versus weekend groupings.
  • ● Retained only records with a valid member_casual label.

Feature engineering snippet adding ride length and weekday labels to the combined dataset.

Phase 3 — Summary statistics and trend analysis

With a clean dataset, I summarized behavior by rider type, weekday and season. These statistics reveal that casual riders take longer trips concentrated on weekends while members ride more often on weekdays.

3.1 — Comparing ride duration and volume by rider type

Using group_by() and summarise(), I computed mean, median, min and max ride length for each rider segment along with total ride counts.

  • ● Casual riders show longer average ride durations than members.
  • ● Members complete more rides overall despite shorter individual trips.
  • ● Summary metrics serve as inputs to dashboard KPIs and narratives.

Summary statistics in R showing relative ride length and volume for casual and member riders.

Additional aggregations by weekday and month surfaced clear seasonality: casual riders cluster in warmer months and on weekends, while member activity looks more like a workweek commute pattern.

  • ● Built weekday summaries for ride counts and average duration.

Sample R scripts — from raw CSVs to enriched dataset

These snippets show how the analysis flows from package setup through import, cleaning, summary statistics and plotting. They mirror the structure of the Cyclistic case study so that another analyst can reproduce the work.

Package setup and import 

install.packages(c("tidyverse","lubridate","janitor","ggplot2","readr"))

library(tidyverse)
library(lubridate)
library(janitor)

file_list <- list.files("data_raw", pattern = "*.csv", full.names = TRUE)

all_trips <- file_list %>%
  map_df(read_csv) %>%
  clean_names()

Cleaning, features and summary 

all_trips <- all_trips %>%
  mutate(
    started_at  = ymd_hms(started_at),
    ended_at    = ymd_hms(ended_at),
    ride_length = as.numeric(difftime(ended_at, started_at, units = "mins")),
    day_of_week = wday(started_at, label = TRUE)
  ) %>%
  filter(ride_length > 1, !is.na(member_casual)) %>%
  drop_na()

summary_stats <- all_trips %>%
  group_by(member_casual) %>%
  summarise(
    average_ride_length = mean(ride_length),
    median_ride_length  = median(ride_length),
    max_ride_length     = max(ride_length),
    min_ride_length     = min(ride_length),
    ride_count          = n()
  )

Membership & Behavior Highlights

6 KEY TAKEAWAYS
WEEKEND LEISURE RIDERS

Casual usage spikes on weekends and in warmer months, suggesting that many casuals view Cyclistic as a leisure or tourism service.

WEEKDAY COMMUTE PATTERNS

Members ride more consistently on weekdays, with shorter, repeatable trips that align with commuting or errand behavior.

DURATION GAP

Casual riders spend nearly twice as long on each ride as members, indicating different expectations and use cases.

CONVERSION OPPORTUNITY

High frequency casual riders look similar to members in total rides and could be targeted with tailored membership offers.

SEASONALITY AWARE PRICING

Promotions timed to peak casual months can amplify ridership while nudging repeat users toward discounted memberships.

DASHBOARD AS STORYTELLING

The Tableau dashboard turns millions of trip records into a simple story about when and how people ride, supporting both marketing and operations decisions.

Growth and Retention Strategy

Tier 1: Acquisition

Objective: Convert high usage casual riders into entry level members.

  • • Weekend membership promos Offer time limited discounts or free days for riders who take multiple weekend trips.
  • • In app conversion nudges Trigger messages like “You have taken 3 rides this week — save with a membership”.
  • • Tourist bundles Create day or weekend bundles that introduce casual riders to membership benefits.

Tier 2: Retention

Objective: Keep members engaged through consistent value and convenience.

  • • Commute friendly features Highlight dock availability, route suggestions and time savings for weekday trips.
  • • Usage based rewards Offer perks for members who hit monthly ride milestones.
  • • Churn early warning Monitor drops in member usage and trigger retention campaigns before renewals lapse.

Tier 3: Operations

Objective: Use behavioral data to keep the system efficient and rider friendly.

  • • Bike redistribution planning Align rebalancing with known weekday and weekend hot spots.
  • • Demand forecasting Use historical patterns to anticipate high traffic days and station congestion.
  • • Continuous KPI monitoring Track ride volume, duration and member mix as ongoing health indicators.

Key insights

  • Casual riders show strong weekend and warm season peaks, reflecting leisure and tourism usage.
  • Members ride more consistently on weekdays with shorter, utility focused trips that resemble commuting.
  • Average ride duration for casual riders is nearly double that of members, signaling different expectations and price sensitivity.
  • Segmentation by rider type, weekday and season creates clear levers for targeted membership campaigns.
  • Combining R and Tableau provides a repeatable pattern for transforming raw open data into decisions.
