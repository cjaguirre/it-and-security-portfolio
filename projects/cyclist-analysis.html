Project Narrative

This project used 12 months of Cyclistic (Divvy) trip data from Chicago to uncover behavioral differences between casual riders and annual members. Using R (tidyverse, lubridate, janitor, ggplot2) for preparation and analysis plus Tableau for visualization, I explored ride volume, duration, and timing patterns across seasons and weekdays.

Insights directly support membership growth strategies by identifying how casual riders behave compared to members and where targeted promotions or in-app nudges can convert high-usage casuals into long-term subscribers.

What I Built

  • Merged and cleaned 12 months of Divvy trip data in R using tidyverse.
  • Engineered ride_length and day_of_week features for segmentation.
  • Computed summary statistics to compare ride behavior between rider types.
  • Developed ggplot2 visuals and a Tableau dashboard to surface trends and KPIs.
  • Produced concrete recommendations to increase membership conversion.

2. Data Import & Standardization

Merge 12 months of trip data using tidyverse

The analysis began by downloading and unzipping 12 monthly CSV files. These were imported with read_csv() and combined into a single dataframe using map_df() and bind_rows(), with clean_names() used to standardize column names like ride_id, rideable_type, started_at, ended_at, and member_casual.

3. Data Cleaning & Feature Engineering

ride_length & day_of_week engineered with lubridate

Cleaning removed rides with negative or zero duration, missing rider types, or incomplete timestamps. Using lubridate, timestamps were parsed and two key features were created: numeric ride length in minutes and labeled day of week for behavior comparison.

  • Converted started_at and ended_at with ymd_hms().
  • Computed ride_length as the difference in minutes between timestamps.
  • Derived day_of_week labels for weekday vs weekend analysis.
  • Filtered out rides with ride_length <= 1 or missing member_casual.

Data Import, Cleaning, and Feature Engineering – R Script 

library(tidyverse)
library(lubridate)
library(janitor)

# Import and combine 12 monthly CSVs
file_list <- list.files("data_raw", pattern = "*.csv", full.names = TRUE)

all_trips <- file_list %>%
  map_df(read_csv) %>%
  clean_names()

# Clean and engineer features
all_trips <- all_trips %>%
  mutate(
    started_at = ymd_hms(started_at),
    ended_at   = ymd_hms(ended_at),
    ride_length = as.numeric(difftime(ended_at, started_at, units = "mins")),
    day_of_week = wday(started_at, label = TRUE)
  ) %>%
  filter(ride_length > 1, !is.na(member_casual)) %>%
  drop_na()

4. Summary Statistics & Trend Analysis

Behavioral differences between casual & member riders

Using dplyr, the cleaned data was grouped by rider type to calculate summary metrics. This highlighted that casual riders take longer trips on average, while members take shorter but more frequent rides.

  • Computed mean, median, min, and max ride length per rider type.
  • Counted total rides for casual vs member segments.
  • Created weekday and seasonal summaries to capture when each group rides most.

Summary Statistics by Rider Type – R Script 

summary_stats <- all_trips %>%
  group_by(member_casual) %>%
  summarise(
    average_ride_length = mean(ride_length),
    median_ride_length  = median(ride_length),
    max_ride_length     = max(ride_length),
    min_ride_length     = min(ride_length),
    ride_count          = n()
  )

print(summary_stats)

5. Visualizing Rider Behavior

ggplot2 charts for weekday patterns & duration gaps

Visual analysis combined ggplot2 for R-based exploration and Tableau for interactive, stakeholder-friendly dashboards. Charts focused on ride volume, duration, and weekday patterns by rider type.

Chart A – Number of Rides by Day of Week

This bar chart shows ride volume by weekday for casual and member riders. Casual usage spikes on Saturdays and Sundays, while member rides peak Monday–Friday, reflecting commuting patterns.

Chart B – Average Ride Duration by Day of Week

Average ride duration by weekday shows that casual riders stay out longer, especially on weekends, while members maintain shorter, consistent trip lengths throughout the week.

Chart C – Average Ride Duration by Rider Type

Aggregated across the full year, casual riders average roughly 23–24 minutes per ride, while members average about 12–13 minutes. This gap reinforces that members typically use the service for short, utility-focused trips such as commuting.

Cyclistic Rider Behavior Dashboard (Tableau)

The Tableau dashboard summarizes 12 months of rider behavior, comparing casual and member riders with filters for day of week, hour of day, and rider type. It is designed for non-technical marketing stakeholders to explore patterns and test “what-if” scenarios.

Dashboard Overview

  • Purpose: Analyze rider behavior to inform membership growth strategies.
  • Timeframe: April 2022 – March 2023.
  • Key filters: Rider Type (Casual vs Member), Day of Week, Hour of Day.

Key Performance Indicators

  • Total Rides: 4,172,559
  • Member Rides: 2,319,482
  • Casual Rides: 1,853,077
  • Average Duration (Casual): ~26.7 minutes
  • Average Duration (Member): ~12.5 minutes

Key Insights

  • Casual riders primarily ride on weekends and in warmer months, reflecting leisure use.
  • Members are more active on weekdays, with shorter but more frequent rides suggesting commuting behavior.
  • Duration gaps between groups reinforce the difference between utility and recreational usage.

Strategic Recommendations

  • Launch weekend-focused promotions for casual riders (discounted passes, referral incentives).
  • Highlight reliability, convenience, and cost savings of membership for frequent weekday riders.
  • Use in-app prompts to nudge casual riders toward membership after repeated usage (for example: “You’ve taken 3 rides this week — save with a membership.”).
