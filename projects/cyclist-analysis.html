What I built

Combined 12 months of Cyclistic trip data in R using tidyverse, lubridate, and ggplot2 — cleaning, engineering features, and summarizing behavioral patterns. Then piped the results into a Tableau dashboard that gives marketing a clear, filterable view of how different riders use the system.

Key outcomes

  • • 12 monthly CSVs merged and cleaned into one 4.2M-row dataset
  • • ride_length and day_of_week features engineered for segmentation
  • • Clear behavior gap: casual riders take 2× longer trips, concentrated on weekends
  • • Tableau dashboard with filters for rider type, weekday, and hour
  • • Concrete membership conversion recommendations for marketing

Phases 1–2 — Import & Cleaning

The 12 monthly CSVs are downloaded from the Divvy portal and stacked into one dataframe in a single pipeline call. Column names get normalized so everything is consistent before cleaning begins. The main cleaning steps handle timestamps, remove impossible durations, and drop incomplete rows.

Import code

library(tidyverse)
library(lubridate)
library(janitor)

file_list <- list.files("data_raw",
                        pattern = "*.csv")

all_trips <- file_list %>%
  map_df(read_csv) %>%
  clean_names()

Cleaning steps

  • • Timestamps parsed with ymd_hms()
  • • Rides ≤ 1 minute filtered out (docking errors)
  • • Rides > 5 days filtered out (outliers / lost bikes)
  • • Rows with missing values dropped
  • • Only valid member_casual labels retained

RStudio console confirming packages installed and ready

Phase 3 — Feature Engineering

The raw data doesn't have a ride duration column or a day-of-week label — both need to be derived. These two features are what make the behavioral comparison between rider types possible.

Engineering code

all_trips <- all_trips %>%
  mutate(
    ride_length = difftime(
      ended_at, started_at
    ),
    day_of_week = wday(
      started_at, label = TRUE
    )
  ) %>%
  filter(ride_length > 1) %>%
  drop_na()

Why these features matter

  • ride_lengthEnables the core comparison — casual riders take 2× longer trips on average. Without this field, the duration pattern is invisible.
  • day_of_weekSeparates weekday commute behavior from weekend leisure use — the clearest signal for conversion targeting.

Phase 4 — Analysis & ggplot2 Charts

With clean, enriched data, behavior was summarized by rider type, weekday, and month. The key finding: casual riders take longer, weekend-concentrated trips while members ride shorter, weekday-consistent routes — a pattern that maps directly to conversion strategy.

Summary statistics

  • • Casual avg: 26.7 min per trip
  • • Member avg: 12.5 min per trip — about half as long
  • • Members complete more total rides: 2.32M vs 1.85M casual
  • • Casual peaks on Sat–Sun; member volume peaks Mon–Fri

Tier 1 — Acquisition

Convert high-usage casual riders into entry-level members.

  • Weekend membership promosTime-limited discounts for riders who take multiple weekend trips — targets the exact casual usage pattern in the data.
  • In-app conversion nudgesTrigger messaging like "You've taken 3 rides this week — save with a membership."
  • Tourist bundlesDay or weekend passes that introduce casual riders to the membership value proposition.

Tier 2 — Retention

Keep members engaged through value and convenience.

  • Commute-friendly featuresHighlight dock availability and time savings for the weekday usage pattern members already show.
  • Usage-based rewardsPerks for members who hit monthly ride milestones reinforce the habit.
  • Churn early warningMonitor drops in member ride frequency and trigger retention campaigns before they lapse.

Tier 3 — Operations

Use behavioral data to keep the system running smoothly.

  • Bike redistributionAlign rebalancing with known weekday and weekend hotspots from the data.
  • Demand forecastingHistorical seasonality patterns anticipate high-traffic days and congestion points.
  • Ongoing KPI monitoringTrack ride volume, duration, and member mix as continuous health indicators.

What I Learned

Key findings

  • • Casual riders take nearly twice as long per trip as members — a consistent signal across all months
  • • Casual usage peaks on weekends and in summer; member usage is stable across weekdays year-round
  • • These two patterns create natural conversion levers: weekend promos and seasonal campaigns
  • • Segmenting by rider type, weekday, and season is enough to build actionable marketing targeting
  • • R for exploration + Tableau for communication is a repeatable, scalable workflow for this kind of analysis
