Cyclistic Rider Behavior Analysis

A twelve-month analysis of Chicago’s Cyclistic bike-share program, comparing how casual riders and annual members differ in seasonality, trip duration, behavior, and peak usage. Insights support marketing strategies focused on converting casual riders into long-term members.

Rider Segmentation Usage Trends

Project Narrative

This project explores how casual riders and Cyclistic annual members behave differently across seasons and times of day. By combining twelve months of trip data in R and visualizing findings in Tableau, the analysis highlights clear differences in trip duration, usage patterns, and preferred ride types.

These behavioral insights help Cyclistic target marketing campaigns toward high-conversion segments during their peak activity windows.

What I Built

  • Merged and cleaned 12 monthly datasets using R.
  • Calculated ride duration, weekdays, seasons, and usage segments.
  • Compared casual vs. member behavior with summary statistics.
  • Developed Tableau dashboards for seasonal, hourly, and geographic trends.
  • Delivered insights aligned with Cyclistic’s membership-growth strategy.

Dataset & Tools

The analysis used over five million Cyclistic trips across twelve months, focusing on timestamps, rideable types, station locations, and user type (member or casual). R handled cleaning and wrangling, while Tableau powered interactive exploration.

  • 12 months of Cyclistic trip data (>5 million rides)
  • Trip durations, bike types, start/end timestamps
  • R for combining and transforming datasets
  • Tableau for dashboards and rider segmentation visuals

Key Findings

  • Casual riders peak on weekends, especially during warm afternoon hours.
  • Members show consistent weekday usage tied to commuting routines.
  • Electric bikes are favored by casual riders for recreational trips.
  • Casual riders take longer rides that follow scenic or leisure routes.
  • Members take shorter, utility-driven trips aligned with errands or work.

Sample R Code

library(tidyverse)

# Load and combine monthly files
files <- list.files(pattern = "*.csv")
all_trips <- files %>% 
  map_df(~ read_csv(.))

# Clean durations and remove bad data
clean_trips <- all_trips %>% 
  mutate(ride_length = ended_at - started_at,
         day_of_week = weekdays(started_at)) %>%
  filter(ride_length > 0)

# Summary comparison
clean_trips %>% 
  group_by(member_casual) %>% 
  summarise(
    avg_duration = mean(ride_length),
    total_rides = n()
  )
