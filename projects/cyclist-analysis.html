Cyclistic Rider Behavior Analysis

A year-long analysis of Chicago’s bike-share program comparing casual riders and annual members. Findings support the Google Data Analytics Capstone objective: informing strategies to convert high-usage casual riders into long-term members.

Rider Segmentation Usage Trends

Project Narrative

This project analyzes 12 months of Cyclistic bike-share trip data to understand differences between casual riders and members. Using R for data wrangling and Tableau for visualization, I identified behavior patterns tied to seasonality, ride duration, and weekday vs weekend usage.

Insights from this analysis drive membership-focused marketing campaigns, in alignment with the Google Data Analytics Capstone workflow: Ask → Prepare → Process → Analyze → Share → Act.

What I Built

  • Merged and cleaned 12 months of trip files using R (tidyverse, lubridate).
  • Created ride_length and day_of_week features for behavioral segmentation.
  • Performed group-by statistical summaries comparing user types.
  • Built Tableau dashboards for seasonal, weekday, and hourly usage patterns.
  • Delivered insights targeted toward converting casual riders to members.

Sample R Code

library(tidyverse)
library(lubridate)

# Load and combine monthly files
files <- list.files(pattern = "*.csv")
all_trips <- files %>% 
  map_df(read_csv) %>% 
  clean_names()

# Clean durations and remove bad data
all_trips <- all_trips %>% 
  mutate(
    started_at = ymd_hms(started_at),
    ended_at = ymd_hms(ended_at),
    ride_length = as.numeric(difftime(ended_at, started_at, units = "mins")),
    day_of_week = wday(started_at, label = TRUE)
  ) %>% 
  filter(ride_length > 1, !is.na(member_casual))

# Summary comparison
summary_stats <- all_trips %>% 
  group_by(member_casual) %>% 
  summarise(
    average_ride_length = mean(ride_length),
    ride_count = n()
  )
